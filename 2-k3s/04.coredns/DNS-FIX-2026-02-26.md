# DNS Fix Summary - February 26, 2026

> **Historical document.** The `*.internal.epaflix.com` domain is no longer used for K3s services. All services now use `*.epaflix.com` directly. This document is kept for reference only.

## Problems Encountered

### 1. Pods Cannot Resolve DNS Queries
**Symptoms:**
- Seerr and other pods showing continuous DNS lookup failures
- Error messages: `DNS lookup failed for auth.epaflix.com after 3 retries`
- Error messages: `DNS lookup failed for sonarr.internal.epaflix.com after 3 retries`
- Pods falling back to native DNS with no success

**Root Cause:**
The DNS server at 192.168.10.30 (Pi-hole) only accepts queries from the 192.168.10.0/24 network. Pod network (10.42.x.x) queries were being refused or timing out.

### 2. Local Network Cannot Access *.internal.epaflix.com
**Symptoms:**
- Browser error: `SSL_ERROR_NO_CYPHER_OVERLAP` when accessing https://sonarr.internal.epaflix.com
- HTTPS connections failing from local machines

**Root Cause:**
- `*.internal.epaflix.com` is registered in public DNS pointing to Cloudflare IPs
- Local machines were resolving to Cloudflare (104.21.59.155, 172.67.179.219) instead of local Traefik (192.168.10.101)
- Pi-hole wildcard `/*.epaflix.com` was not catching `*.internal.epaflix.com` queries
- DNS cache on local machines was holding onto wrong IPs

## Solutions Implemented

### 1. Configure systemd-resolved on K3s Nodes

**What:** Configure systemd-resolved to listen on node IPs so pods can query DNS via the nodes.

**Implementation:**
```bash
# On all K3s nodes (masters and workers: 51, 52, 53, 61, 62, 63, 65)
for ip in 51 52 53 61 62 63 65; do
  ssh ubuntu@192.168.10.$ip "sudo mkdir -p /etc/systemd/resolved.conf.d/ && sudo tee /etc/systemd/resolved.conf.d/listen.conf > /dev/null << 'EOL'
[Resolve]
DNSStubListenerExtra=192.168.10.$ip
DNSStubListenerExtra=10.0.0.$ip
EOL
sudo systemctl restart systemd-resolved"
done
```

**Verification:**
```bash
ssh ubuntu@192.168.10.51 "sudo ss -tulpn | grep 53"
# Should show systemd-resolved listening on 192.168.10.51:53 and 10.0.0.51:53
```

### 2. Update CoreDNS Main Configuration

**What:** Change CoreDNS to forward queries to node IPs instead of directly to DNS server or /etc/resolv.conf.

**Implementation:**
```bash
kubectl patch configmap coredns -n kube-system --type=json -p='[{"op":"replace","path":"/data/Corefile","value":".:53 {\n    errors\n    health\n    ready\n    kubernetes cluster.local in-addr.arpa ip6.arpa {\n      pods insecure\n      fallthrough in-addr.arpa ip6.arpa\n    }\n    hosts /etc/coredns/NodeHosts {\n      ttl 60\n      reload 15s\n      fallthrough\n    }\n    prometheus :9153\n    cache 30\n    loop\n    reload\n    loadbalance\n    import /etc/coredns/custom/*.override\n    forward . 192.168.10.51 192.168.10.52 192.168.10.53\n}\nimport /etc/coredns/custom/*.server\n"}]'
```

**Key Change:** `forward . 192.168.10.51 192.168.10.52 192.168.10.53` (node IPs) instead of `forward . /etc/resolv.conf` or `forward . 192.168.10.30`

### 3. Update Custom epaflix Domains Configuration

**What:** Update the custom ConfigMap for *.epaflix.com domains to also forward to node IPs.

**Implementation:**
```bash
kubectl patch configmap coredns-custom -n kube-system --type=json -p='[{"op":"replace","path":"/data/epaflix.server","value":"epaflix.com:53 internal.epaflix.com:53 {\n    errors\n    cache 30\n    forward . 192.168.10.51 192.168.10.52 192.168.10.53\n    log\n}\n"}]'
```

**Restart CoreDNS:**
```bash
kubectl rollout restart deployment/coredns -n kube-system
kubectl rollout status deployment/coredns -n kube-system
```

### 4. Add Explicit DNS Record in Pi-hole

**What:** Add specific DNS record for `*.internal.epaflix.com` in Pi-hole to override public DNS.

**Implementation:**
```bash
ssh root@192.168.10.30
cat >> /etc/dnsmasq.d/05-epaflix.conf << 'EOF'

# Explicitly set *.internal.epaflix.com to local Traefik
address=/*.internal.epaflix.com/192.168.10.101
EOF

pihole reloaddns
```

**File Location:** `/etc/dnsmasq.d/05-epaflix.conf` on Pi-hole (192.168.10.30)

**Final Configuration:**
```
# Route ONLY subdomains to local host
# The root domain epaflix.com will use normal upstream resolution
server=/epaflix.com/

# All other *.epaflix.com subdomains go to local Traefik
address=/*.epaflix.com/192.168.10.101

# Explicitly set *.internal.epaflix.com to local Traefik
address=/*.internal.epaflix.com/192.168.10.101
```

### 5. Clear DNS Cache on Local Machines

**What:** Flush DNS cache on local machines to remove cached Cloudflare IPs.

**Implementation:**
```bash
# Linux with systemd-resolved
sudo systemd-resolve --flush-caches

# Verify resolution
dig sonarr.internal.epaflix.com +short
# Should return: 192.168.10.101
```

## Technical Details

### DNS Resolution Flow (After Fix)

**For Pods:**
```
Pod → CoreDNS (10.43.0.10) → Node IPs (192.168.10.51-53:53) → systemd-resolved → Pi-hole (192.168.10.30) → Upstream DNS
```

**For *.epaflix.com domains (from pods):**
```
Pod → CoreDNS custom server block → Node IPs → systemd-resolved → Pi-hole → Returns 192.168.10.101
```

**For Local Machines:**
```
Application → systemd-resolved → Pi-hole (192.168.10.30) → Returns 192.168.10.101 (from dnsmasq wildcard)
```

### Why This Works

1. **Nodes as DNS Proxies:** Nodes have systemd-resolved listening on their IPs, which accepts queries from any network (including pod network).

2. **Pi-hole ACL:** Pi-hole only accepts queries from 192.168.10.0/24. Since queries now come from node IPs (192.168.10.51-53), they're accepted.

3. **Explicit Wildcards:** Added explicit `/*.internal.epaflix.com/192.168.10.101` in Pi-hole to override public DNS records.

4. **CoreDNS Forwarding:** CoreDNS forwards to multiple node IPs for redundancy and load balancing.

## Troubleshooting

### If DNS stops working for pods:
1. Check systemd-resolved is listening on node IPs: `ssh ubuntu@192.168.10.51 "sudo ss -tulpn | grep 53"`
2. Check CoreDNS is forwarding to node IPs: `kubectl get configmap coredns -n kube-system -o yaml | grep forward`
3. Restart CoreDNS: `kubectl rollout restart deployment/coredns -n kube-system`

### If local machines can't resolve *.internal.epaflix.com:
1. Verify Pi-hole has the record: `ssh root@192.168.10.30 "dig @127.0.0.1 sonarr.internal.epaflix.com +short"`
2. Clear local DNS cache: `sudo systemd-resolve --flush-caches`
3. Check Pi-hole dnsmasq config: `ssh root@192.168.10.30 "cat /etc/dnsmasq.d/05-epaflix.conf"`
4. Reload Pi-hole DNS: `ssh root@192.168.10.30 "pihole reloaddns"`
